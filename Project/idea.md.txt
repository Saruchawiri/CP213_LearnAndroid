## Project Proposal  
## Mobile Application: Whatever in BKK  

---

## Step 1: Project Overview  

### 1. ชื่อโครงการ  
**Whatever in BKK**  
แอปพลิเคชันช่วยตัดสินใจเรื่องเล็ก ๆ ในชีวิตประจำวัน  

---

### 2. ที่มาและความสำคัญของโครงการ  
ในชีวิตประจำวัน ผู้คนมักพบกับปัญหาในการตัดสินใจเรื่องง่าย ๆ เช่น  

- วันนี้จะกินอะไรดี  
- จะไปไหนดี  
- ควรเริ่มทำงานชิ้นไหนก่อน  

แม้จะเป็นเรื่องเล็ก แต่กลับใช้เวลาในการตัดสินใจค่อนข้างนาน โดยเฉพาะเมื่ออยู่เป็นกลุ่ม ทำให้เกิดความล่าช้าและเสียเวลาโดยไม่จำเป็น  

ดังนั้น แอป **Whatever in BKK** จึงถูกพัฒนาขึ้นเพื่อช่วยให้การตัดสินใจเป็นเรื่องง่ายและรวดเร็ว โดยให้ผู้ใช้สามารถเพิ่มตัวเลือกของตนเอง และให้ระบบสุ่มคำตอบทันที  

นอกจากนี้ แอปยังเพิ่มฟีเจอร์ **Mood-based Random (สุ่มตามอารมณ์)** เพื่อช่วยให้การตัดสินใจมีความเหมาะสมกับความรู้สึกของผู้ใช้มากยิ่งขึ้น เช่น  

- ❤️ Romance → ร้านอาหารบรรยากาศดี / ดินเนอร์ / วิวสวย  
- 🌿 Relax → คาเฟ่เงียบ ๆ / ธรรมชาติ / นั่งชิล  
- 🎉 Fun → ร้านคึกคัก / แฮงเอาท์ / กิจกรรมสนุก  
- 😔 Depressed → ร้านฮีลใจ / ของหวาน / สถานที่สงบ  

ซึ่งช่วยเพิ่มประสบการณ์การใช้งานให้มีความฉลาดและตรงใจผู้ใช้มากขึ้น  

---

### 3. วัตถุประสงค์  

- เพื่อพัฒนา Mobile Application บน Android สำหรับช่วยตัดสินใจในชีวิตประจำวัน  
- เพื่อฝึกพัฒนาแอปด้วย Kotlin และ Jetpack Compose  
- เพื่อฝึกการออกแบบระบบด้วย MVVM Architecture เบื้องต้น  
- เพื่อเรียนรู้การจัดเก็บข้อมูลในเครื่อง (Local Storage)  
- เพื่อพัฒนาฟีเจอร์การสุ่มแบบอิงอารมณ์ (Mood-based Recommendation)  

---

### 4. กลุ่มเป้าหมาย  

- นักศึกษา  
- คนวัยทำงาน (อายุ 18–30 ปี)  
- กลุ่มเพื่อนที่ตัดสินใจยาก  
- ผู้ที่ต้องการแอปใช้งานง่าย รวดเร็ว และไม่ซับซ้อน  

---

## Step 2: Tech Stack  

### 2.1 Platform  

- Android Application  
- Android Studio  
- Kotlin  

---

### 2.2 UI Development  

- Jetpack Compose  
- Simple / Clean UI  
- State-based (Reactive UI)  

---

### 2.3 Architecture  

- MVVM (Model – View – ViewModel)  
  - Model → จัดการข้อมูล  
  - View (Compose UI) → แสดงผล  
  - ViewModel → ควบคุม Logic และ State  

---

### 2.4 Navigation  

- Multi-Activity  
- ใช้ `startActivity()`  
- เพื่อฝึก Android Activity Lifecycle  

**Screens:**  
- MainActivity → เพิ่มตัวเลือก  
- ResultActivity → แสดงผลลัพธ์  
- HistoryActivity → ประวัติ  

---

### 2.5 Database & Storage  

- Room Database (SQLite) → เก็บ History  
- Offline Ready  

**Fallback:**  
- SharedPreferences (กรณีฉุกเฉิน)  

---

### 2.6 Offline Capability  

- ใช้งานได้โดยไม่ต้องใช้อินเทอร์เน็ต  
- เก็บข้อมูลในเครื่องผู้ใช้  

---

## Step 3: System Design  

### 3.1 แนวคิดการออกแบบ  

- Minimal  
- User-Friendly  
- รองรับการเลือก “Mood” ก่อนสุ่ม  

---

### 3.2 Screen Structure  

**Main Screen (MainActivity)**  

- Input ตัวเลือก  
- ปุ่ม Add  
- List ตัวเลือก  
- Mood Selector (Chips / Buttons):  
  - ❤️ Romance  
  - 🌿 Relax  
  - 🎉 Fun  
  - 😔 Depressed  
- ปุ่ม Decide  

---

**Result Screen (ResultActivity)**  

- แสดงผลลัพธ์  
- แสดง Mood ที่เลือก  
- Animation  
- ปุ่ม Try Again / Back  

---

**History Screen (HistoryActivity)**  

- รายการย้อนหลัง  
- แสดง “ผลลัพธ์ + Mood”  
- ปุ่ม Clear History  

---

### 3.3 System Flow  

1. ผู้ใช้เพิ่มตัวเลือก (≥ 2)  
2. ผู้ใช้เลือก Mood (optional)  
3. กด Decide  
4. ระบบสุ่มโดยอิง Mood  
5. แสดงผลลัพธ์  
6. บันทึก History (พร้อม Mood)  
7. ดูย้อนหลัง / เริ่มใหม่  

---

### 3.4 Data Structure  

**Option**  

- id  
- name  
- moodTag (Romance / Relax / Fun / Depressed)  

**History**  

- id  
- result  
- mood  
- timestamp  

---

## Step 4: Functional Requirements (MVP)  

### 4.1 Add Options  

- เพิ่มตัวเลือก  
- ลบตัวเลือก  
- ต้องมี ≥ 2 ตัวเลือก  

---

### 4.2 Random Decision  

- ปุ่ม Decide  
- สุ่ม 1 ตัวเลือก  
- รองรับการสุ่มตาม Mood  
- แสดงผลชัดเจน  
- มี Animation  

---

### 4.3 Mood-based Random ⭐  

- ผู้ใช้สามารถเลือก Mood ก่อนสุ่ม  
- ระบบจะเลือกเฉพาะตัวเลือกที่ตรงกับ Mood  
- หากไม่มี Mood → สุ่มปกติ  
- รองรับ 4 Mood: Romance, Relax, Fun, Depressed  

---

### 4.4 History  

- บันทึกผลลัพธ์ + Mood  
- แสดงย้อนหลัง  
- ล้างข้อมูลได้  

---

### 4.5 Offline Usage  

- ใช้งานได้โดยไม่ต้องใช้อินเทอร์เน็ต  
- เก็บข้อมูลในเครื่อง  

---

## Step 5: Non-Functional Requirements  

- ใช้งานง่าย  
- ตอบสนองเร็ว  
- Offline ได้  
- UI อ่านง่าย  
- รองรับการใช้งานแบบ Personal มากขึ้น  

---

## Step 6: Expected Outcomes  

- ได้แอป Android ใช้งานจริง  
- เข้าใจ Jetpack Compose  
- เข้าใจ MVVM  
- ใช้ Local Storage ได้  
- ได้เรียนรู้การออกแบบฟีเจอร์ที่ตอบโจทย์ผู้ใช้มากขึ้น  

---

## 💡 Highlight Idea  

> แอปนี้ไม่ใช่แค่ Random ธรรมดา แต่เป็น **Emotion-driven Decision Making**  
> ที่ช่วยให้การตัดสินใจ “ตรงกับอารมณ์ (Romance / Relax / Fun / Depressed)” ของผู้ใช้มากขึ้น
