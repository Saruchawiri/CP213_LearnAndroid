## Project Proposal  
## Mobile Application: Whatever in BKK  

---

## Step 1: Project Overview  

### 1. ชื่อโครงการ  
**Whatever in BKK**  
แอปพลิเคชันช่วยตัดสินใจเรื่องเล็ก ๆ ในชีวิตประจำวัน  

---

### 2. ที่มาและความสำคัญของโครงการ  
ในชีวิตประจำวัน ผู้คนมักพบกับปัญหาในการตัดสินใจเรื่องง่าย ๆ เช่น  
- วันนี้จะกินอะไรดี  
- จะไปไหนดี  
- ควรเริ่มทำงานชิ้นไหนก่อน  

แม้จะเป็นเรื่องเล็ก แต่กลับใช้เวลาในการตัดสินใจค่อนข้างนาน โดยเฉพาะเมื่ออยู่เป็นกลุ่ม ทำให้เกิดความล่าช้าและเสียเวลาโดยไม่จำเป็น  

ดังนั้น แอป **Whatever in BKK** จึงถูกพัฒนาขึ้นเพื่อช่วยให้การตัดสินใจเป็นเรื่องง่ายและรวดเร็ว โดยให้ผู้ใช้สามารถเพิ่มตัวเลือกของตนเอง และให้ระบบสุ่มคำตอบทันที พร้อมเพิ่มความสนุกผ่านการใช้งาน  

---

### 3. วัตถุประสงค์  
- เพื่อพัฒนา Mobile Application บน Android สำหรับช่วยตัดสินใจในชีวิตประจำวัน  
- เพื่อฝึกพัฒนาแอปด้วย Kotlin และ Jetpack Compose  
- เพื่อฝึกการออกแบบระบบด้วย MVVM Architecture เบื้องต้น  
- เพื่อเรียนรู้การจัดเก็บข้อมูลในเครื่อง (Local Storage)  

---

### 4. กลุ่มเป้าหมาย  
- นักศึกษา  
- คนวัยทำงาน (อายุ 18–30 ปี)  
- กลุ่มเพื่อนที่ตัดสินใจยาก  
- ผู้ที่ต้องการแอปใช้งานง่าย รวดเร็ว และไม่ซับซ้อน  

---

## Step 2: Tech Stack  

### 2.1 Platform  
- Android Application  
- Android Studio  
- Kotlin  

---

### 2.2 UI Development  
- Jetpack Compose  
- Simple / Clean UI  
- State-based (Reactive UI)  

---

### 2.3 Architecture  
- MVVM (Model – View – ViewModel)  
  - Model → จัดการข้อมูล  
  - View (Compose UI) → แสดงผล  
  - ViewModel → ควบคุม Logic และ State  

---

### 2.4 Navigation  
- Multi-Activity  
- ใช้ `startActivity()`  
- เพื่อฝึก Android Activity Lifecycle  

**Screens:**  
- MainActivity → เพิ่มตัวเลือก  
- ResultActivity → แสดงผลลัพธ์  
- HistoryActivity → ประวัติ  

---

### 2.5 Database & Storage  
- Room Database (SQLite) → เก็บ History  
- Offline Ready  

**Fallback:**  
- SharedPreferences (กรณีฉุกเฉิน)  

---

### 2.6 Offline Capability  
- ใช้งานได้โดยไม่ต้องใช้อินเทอร์เน็ต  
- เก็บข้อมูลในเครื่องผู้ใช้  

---

## Step 3: System Design  

### 3.1 แนวคิดการออกแบบ  
- Minimal  
- User-Friendly  
- สีสันเล็กน้อย  

---

### 3.2 Screen Structure  

**Main Screen (MainActivity)**  
- Input ตัวเลือก  
- ปุ่ม Add  
- List ตัวเลือก  
- ปุ่ม Decide  

**Result Screen (ResultActivity)**  
- แสดงผลลัพธ์  
- Animation  
- ปุ่ม Try Again / Back  

**History Screen (HistoryActivity)**  
- รายการย้อนหลัง  
- ปุ่ม Clear History  

---

### 3.3 System Flow  
1. ผู้ใช้เพิ่มตัวเลือก (≥ 2)  
2. กด Decide  
3. ระบบสุ่มผลลัพธ์  
4. แสดงผล  
5. บันทึก History  
6. ดูย้อนหลัง / เริ่มใหม่  

---

### 3.4 Data Structure  

**Option**  
- id  
- name  

**History**  
- id  
- result  
- timestamp  

---

## Step 4: Functional Requirements (MVP)  

### 4.1 Add Options  
- เพิ่มตัวเลือก  
- ลบตัวเลือก  
- ต้องมี ≥ 2 ตัวเลือก  

---

### 4.2 Random Decision  
- ปุ่ม Decide  
- สุ่ม 1 ตัวเลือก  
- แสดงผลชัดเจน  
- มี Animation  

---

### 4.3 History  
- บันทึกผลลัพธ์  
- แสดงย้อนหลัง  
- ล้างข้อมูลได้  

---

### 4.4 Offline Usage  
- ใช้งานได้โดยไม่ต้องใช้อินเทอร์เน็ต  
- เก็บข้อมูลในเครื่อง  

---

## Step 5: Non-Functional Requirements  
- ใช้งานง่าย  
- ตอบสนองเร็ว  
- Offline ได้  
- UI อ่านง่าย  

---

## Step 6: Expected Outcomes  
- ได้แอป Android ใช้งานจริง  
- เข้าใจ Jetpack Compose  
- เข้าใจ MVVM  
- ใช้ Local Storage ได้
