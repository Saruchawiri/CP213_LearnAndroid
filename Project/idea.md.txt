Project Proposal
Mobile Application: Whatever in BKK

1. ชื่อโครงการ
Whatever in BKK
แอปพลิเคชันช่วยตัดสินใจเรื่องเล็ก ๆ ในชีวิตประจำวัน

2. ที่มาและความสำคัญของโครงการ
ในชีวิตประจำวัน คนส่วนใหญ่มักลังเลกับเรื่องง่าย ๆ เช่น
วันนี้จะกินอะไรดี
จะไปไหนดี
จะทำงานชิ้นไหนก่อน
บางครั้งการตัดสินใจใช้เวลานาน โดยเฉพาะเมื่ออยู่กับเพื่อน ทำให้เสียเวลาโดยไม่จำเป็น

แอป Whatever in BKK จึงถูกพัฒนาขึ้นเพื่อช่วยตัดสินใจแบบง่าย ๆ โดยให้ผู้ใช้ใส่ตัวเลือกเอง แล้วกดปุ่มให้แอปสุ่มเลือกคำตอบให้ทันที ช่วยลดเวลาในการคิด และเพิ่มความสนุกในการตัดสินใจ

3. วัตถุประสงค์
เพื่อพัฒนาแอป Android ที่ช่วยตัดสินใจในชีวิตประจำวัน
เพื่อฝึกพัฒนาแอปด้วย Kotlin และ Jetpack Compose
เพื่อฝึกการใช้ MVVM Architecture เบื้องต้น
เพื่อเรียนรู้การเก็บข้อมูลในเครื่องด้วย Room หรือ SharedPreferences

4. กลุ่มเป้าหมาย
นักศึกษา
คนวัยทำงานช่วงอายุ 18–30 ปี
กลุ่มเพื่อนที่ตัดสินใจยาก
ผู้ที่ต้องการแอปใช้งานง่ายและรวดเร็ว

5. ขอบเขตของระบบ (MVP)
5.1 เพิ่มตัวเลือก (Add Options)
ผู้ใช้สามารถพิมพ์ตัวเลือกเองได้
ลบตัวเลือกได้
ต้องมีอย่างน้อย 2 ตัวเลือกก่อนกดสุ่ม

5.2 สุ่มเลือก (Random Decision)
มีปุ่ม “Decide”
แอปสุ่มเลือก 1 ตัวเลือกจากรายการ
แสดงผลลัพธ์ชัดเจนบนหน้าจอ
มี animation แบบง่ายเพื่อเพิ่มความน่าสนใจ

5.3 ประวัติการสุ่ม (History)
บันทึกผลลัพธ์ที่สุ่มได้
แสดงรายการย้อนหลัง
สามารถล้างประวัติได้

5.4 การใช้งานแบบ Offline
ฟีเจอร์หลักทั้งหมดสามารถใช้งานได้โดยไม่ต้องเชื่อมต่ออินเทอร์เน็ต
ข้อมูลถูกเก็บในเครื่องของผู้ใช้

6. เทคโนโลยีที่ใช้ (Tech Stack)
6.1 Platform
Android Application
พัฒนาโดยใช้ Android Studio
ภาษา Kotlin

6.2 UI Development
ใช้ Jetpack Compose ในการออกแบบหน้าจอ
ออกแบบ UI แบบเรียบง่าย ใช้งานง่าย

6.3 Architecture
ใช้ MVVM (Model–View–ViewModel) แบบพื้นฐาน
Model → จัดการข้อมูล
View → หน้าจอ Compose
ViewModel → จัดการ Logic การสุ่มและควบคุมข้อมูล

6.4 Navigation
ใช้ Multi-Activity
ใช้ startActivity() สำหรับเปลี่ยนหน้า
เพื่อฝึกเรียนรู้ Android Activity Lifecycl

หน้าจอหลักของแอป:
MainActivity → หน้าเพิ่มตัวเลือก
ResultActivity → หน้าแสดงผลลัพธ์
HistoryActivity → หน้าแสดงประวัติ

6.5 Database
ตัวเลือกหลัก (แนะนำ):
ใช้ Room Database (SQLite)
เก็บข้อมูล History

ตัวเลือกสำรอง (กรณีฉุกเฉิน):
ใช้ SharedPreferences
เก็บข้อมูลแบบพื้นฐาน

7. การออกแบบระบบ (Design)
7.1 แนวคิดการออกแบบ
เรียบง่าย
ใช้งานสะดวก
สีสันสดใสเล็กน้อย

7.2 โครงสร้างหน้าจอ
หน้าแรก (Main Screen)
ช่องกรอกตัวเลือก
ปุ่ม Add
รายการตัวเลือก
ปุ่ม Decide
หน้าผลลัพธ์ (Result Screen)

แสดงตัวเลือกที่สุ่มได้
ปุ่ม Try Again
ปุ่ม Back
หน้าประวัติ (History Screen)
แสดงรายการผลย้อนหลัง
ปุ่ม Clear History
